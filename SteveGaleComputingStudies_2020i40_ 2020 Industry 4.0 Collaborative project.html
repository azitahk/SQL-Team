#2020 IP address of Steves server in VM

#host = '192.168.223.240'   # VM DHCP Lease in E223 (OR 241)

#construct json string with measured values inserted   
contentStr='{ "RoomName" : "E223", "Humidity" : "%.2f", "Temperature": "%.2f" }'%(humidityValue,tempValue)  
postStr = 'POST /i40Test/admin/InsertJsonRESTData.php HTTP/1.1\r\n'  
hostStr = 'Host: %s:%s\r\n'%(str(host),str(port))  
contentTypeStr = 'Content-Type: application/json\r\n'  
contentLengthStr = 'Content-Length: %s\r\n\r\n'%str(len(contentStr))  
 
#format HTTP post payload string for sending to the PHP / mySQL server  
payload = postStr + hostStr+ contentTypeStr + contentLengthStr + contentStr

Config MySQL and firewall rules for remote connections on port 3306

https://www.digitalocean.com/community/tutorials/how-to-allow-remote-access-to-mysql

$sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
Change the bind address ‚Äì note security issues ‚Äì this should be a static IP in production

eg.
$ip a
192.168.100.100 - server ip address
change the bind address to 192.168.100.100

Allow connectivity from the remote windows host that will be analyzing data (192.168.100.X windows VM host in my case , if the firewall is enabled )
$sudo ufw allow from remote_IP_address to any port 3306
$sudo ufw allow from 192.168.100.0/24 to any port 3306
$reboot

You will now be anle to make remote connections to MySQL on port 3306 from your windows 10 PC
